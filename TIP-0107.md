## Abstract

Cerberus is an innovative and optional transaction layer designed for the Talos network that introduces a last-resort mechanism to guarantee censorship resistance. By enabling users to attach a Proof-of-Work (PoW) puzzle to their transactions, Cerberus effectively creates an economic incentive for network participants, ensuring that no valid transaction can be permanently censored. This layer reinforces Talos's commitment to being a truly permissionless and uncensorable platform.

## Motivation

The motivation behind the Cerberus protocol stems from the increasing risk of censorship faced by users when engaging in financial transactions in an increasingly centralized digital landscape. By providing an escape hatch for transactions that are at risk of being blocked, Cerberus empowers users, particularly those operating in repressive regimes or facing collusion among validators, to maintain their rights to operate freely and securely within the Talos ecosystem. This aligns with the foundational principles of blockchain technology and the cypherpunk movement, championing user autonomy and resistance to centralized control.

## Specification

The Cerberus protocol will operate as a secondary transaction layer over the existing Talos infrastructure. Users can initiate a transaction as a standard transaction through the Pauli sequencing pipeline. If the transaction is at risk of censorship, users can escalate it via the Nexus wallet, which will append a PoW puzzle solution, creating a Cerberus Transaction. Cerberus Guardians will then monitor these transactions and prioritize their inclusion into blocks, incentivized by the higher fees associated with these transactions. A dynamic difficulty adjustment mechanism will ensure that the PoW remains balanced and resistant to spam while promoting fairness and accessibility.

## Rationale

The rationale for implementing the Cerberus protocol lies in the need to create a resilient transaction layer that protects users from potential censorship by malicious actors or colluding validators. By allowing users to prove their commitment to a transaction through PoW, Cerberus not only enhances the security of the network but also deters censorship attempts by making them economically disadvantageous. This approach reinforces the values of decentralization, autonomy, and transparency that are integral to the blockchain ethos.

## Security Considerations

The Cerberus protocol incorporates several security considerations to prevent abuse and maintain network integrity. Dynamic difficulty adjustment mitigates the risk of spamming the network with low-value transactions. By utilizing an ASIC-resistant PoW algorithm, the protocol promotes decentralization, preventing a single entity from dominating PoW solving. Furthermore, enabling an open set of Guardians, with strong economic incentives, ensures that the protocol is resilient against collusion and can adapt to changes in the network's validation landscape.

## Implementation

The implementation of the Cerberus protocol will follow a phased approach:

1. Research & Design: Define the PoW algorithm and Guardian incentive model.
2. PoW Solver Library: Develop a library for PoW solving for integration into the Nexus wallet.
3. Guardian Node Software: Modify validator clients to include Guardian functionality.
4. Protocol Integration: Implement Cerberus transaction types and logic into the Hephaestus VM.
5. Nexus Wallet Integration: Enable the "Escalate" button to handle Cerberus Transactions.
6. Testnet Simulation: Conduct long-term testing to validate network behavior under load.
7. Mainnet Activation: Rollout of the fully tested Cerberus protocol via a governance vote.

Each phase will have specific deliverables and acceptance criteria to ensure the security and efficacy of the implementation.
